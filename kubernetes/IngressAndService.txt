Для того, что бы опубликовать приложение и дать доступ до него из-вне, использается две обстракции Ingress и Service

Service типы:
	ClusterIp
	NodePort
	LoadBalancer
	ExternalName
	ExternalIPs
	 
ClusterIp - для отладки и наладки внутрикластерных приложений.(Как правило для локального разворачивания)

  ClusterIp---->POD   
  	  \---->POD
	   ---->POD
	
пример:

	
NodePort - Используется с балансировщиком(со своими правилами куда распределять трафик) на входе до Node 
		или с LoadBalancer 
	   у каждого Node создаётся свой Port номер от 30000 до 32768,
	   и трафики будут адресовываться из Node в определенный POD

			Traffic
			/   |  \
		       /    |   \
		    port   port  port
		    Node   Node  Node
		      \     |     /
		       \    |    /
		         Service
		             |----->POD
		             |----->POD
		             ------>POD
		  Service и POD нахятся в Cluster
		  
пример:

LoadBalancer - балансировщик, но используемый как правило только в Облаке 
		и предоставляемый самим поставщиком Облака(можно прописать статический IP)
		
			Traffic
			   |  
		           | 
		    LoadBalancer
		           |     
		           |    
		         Service
		             |----->POD
		             |----->POD
		             ------>POD
		  Service и POD нахятся в Cluster
		  
пример:


ExternalName - прописывается ДНС, для того, что бы POD смог обратиться куда то к севрису,
		а он уже перенаправил его согласно ДНС.


			   Service
			     /|\
		              |<-----POD
		              |<-----POD
		              |<------POD


пример:

ExternalIPs - сервис используется так же с балансировщиком, а балансировщик 
		далее пробрасывает именно на vipIp
   			
   			Traffic
			/   |  \
		       /    |   \
		      Ip   Ip   Ip
		    Node   Node  Node
		      \     |     /
		       \    |    /
		         Service
		             |----->POD
		             |----->POD
		             ------>POD
