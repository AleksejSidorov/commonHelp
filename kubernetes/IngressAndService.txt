Для того, что бы опубликовать приложение и дать доступ до него из-вне, использается две обстракции Ingress и Service(Loadbalacer и NodePort(плохая реализация))
Сервисы используются для изолирования POD, а так же в основном предоставляют DNS для POD. Но, сервисы не являются proxy, а всего лишь являются 
IP TABLE - таблица сопоставлении различные IP, алиесов и разновидных правил для них(к примеру для LoadBalancer прописан рандомайзер, который +- одинаковых пропоциях раскидывает трафик)

Ingress - это уже реальный прокси, со своей app. В качестве app исползуются стороннее по, как правило nginx. И при пробрасывании портов на ружу, следует использовать именно Ingress.

Service типы:
	ClusterIp
	NodePort
	LoadBalancer
	ExternalName
	ExternalIPs
	 
ClusterIp - для отладки и наладки внутрикластерных приложений.(Как правило для локального разворачивания)

  ClusterIp---->POD   
  	  \---->POD
	   ---->POD

NodePort - Используется с балансировщиком(со своими правилами куда распределять трафик) на входе до Node 
		или с LoadBalancer 
	   у каждого Node создаётся свой Port номер от 30000 до 32768,
	   и трафики будут адресовываться из Node в определенный POD

			Traffic
			/   |  \
		       /    |   \
		    port   port  port
		    Node   Node  Node
		      \     |     /
		       \    |    /
		         Service
		             |----->POD
		             |----->POD
		             ------>POD
		  Service и POD нахятся в Cluster
		  
пример:

		           |    
		         Service

HeadLess - Это сервис, который предоставляет DNS без предоставления выделенного IP.
		При обращении к HeadLess нам предоставят IP подов из предоставленных 
		в selector
